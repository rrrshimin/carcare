# CarCare Diary AI Development Guide

This repository contains the documentation and implementation plan for the CarCare Diary mobile application.

The documentation inside /docs is the source of truth for product behavior, architecture, and business logic.

AI coding agents working in this repository must follow the documentation before implementing features.

Primary documents:

- docs/product-overview.md
- docs/app-flows.md
- docs/screen-specifications.md
- docs/business-logic.md
- docs/database-schema.md
- docs/project-structure.md
- docs/implementation-roadmap.md
- docs/testing-and-acceptance-criteria.md
- docs/style-guide.md
- docs/ai-coding-rules.md

Rules:

1. Never invent product features not described in documentation.
2. Follow implementation phases defined in implementation-roadmap.md.
3. Use testing-and-acceptance-criteria.md to validate implemented features.
4. Prefer simple and readable code over complex abstractions.
5. Implement features incrementally and verify each phase before continuing.

When uncertain, prefer the documented behavior rather than assumptions.
